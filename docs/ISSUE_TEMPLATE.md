# ISSUE_TEMPLATE

Use this template for every Phase 1 GitHub issue.

## Title
[easy|medium|hard] Short action-oriented title

## Description
- Explain what changes in code or docs.
- Name impacted files/modules.
- Keep scope to one deliverable.

## Why This Matters
- Explain project-level impact in 1-2 sentences.

## Requirements
- List concrete implementation tasks.
- Prefer file/function-level language.
- Avoid generic statements.

## Prerequisites
- Link dependent issues using #<issue-number> when available.
- If no dependency exists, state None.

## External References
- Add official docs links only when needed.
- If not needed, write None required.

## Acceptance Criteria
- Use Markdown checkboxes (`- [ ]`).
- Make criteria executable and verifiable.
- Include concrete commands/files/expected outcomes.
- Cover at least one failure-path check where relevant.

## Notes
- Do not include Labels section in issue body.
- Do not include Definition of Done section in issue body.
- Labels are managed in GitHub metadata, not Markdown body.
